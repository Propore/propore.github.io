1.4 面试常考业务题

题目1：如何保证消息的顺序性？

关键点：

  • 生产者：相同Key发送到同一分区
  • 消费者：单分区单消费者串行处理
  • 或者：使用单分区 + 单消费者组

题目2：如何保证消息不丢失？

三处确认点：

位置 配置 作用
生产者 acks=all Broker确认写入成功
Broker replication.factor=3 多副本冗余
消费者 手动提交 业务处理完再提交

附录：面试高频问题速记

问题 核心关键词
Kafka 如何保证顺序 生产者：相同Key同分区；消费者：单分区单线程
Kafka 如何保证不丢失 acks=all + retries + 手动提交偏移量
Kafka 如何保证幂等 业务层去重：唯一ID/数据库唯一约束
ES 如何实现分词搜索 IK分词器 + 自定义analyzer
ES 如何优化查询性能 routing + _source过滤 + filter缓存 + search_after
ES 如何实现聚合分析 bucket + metric 嵌套聚合
ES vs MySQL 适用场景 ES：全文搜索、复杂查询、聚合分析；MySQL：事务、强一致性
