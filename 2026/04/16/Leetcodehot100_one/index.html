基础数据结构

1. 数组

原理介绍

数组是最基础的数据结构，它在内存中连续存储相同类型的元素。

  • 优点：通过索引访问元素速度快 O(1)
  • 缺点：插入、删除元素需要移动其他元素 O(n)
  • 适用场景：需要频繁访问元素，元素数量相对固定

Go 语言实现

2. 链表

原理介绍

链表是一种非连续存储的数据结构，每个节点包含数据和指向下一个节点的指针。

  • 优点：插入、删除元素只需改变指针 O(1)
  • 缺点：访问元素需要从头遍历 O(n)
  • 适用场景：频繁插入删除，不需要随机访问

单向链表实现

3. 哈希表

原理介绍

哈希表通过哈希函数将键映射到数组索引，实现快速查找。

  • 原理：key → hash函数 → 数组索引
  • 冲突解决：使用链地址法（数组+链表）
  • 时间复杂度：平均 O(1)，最坏 O(n)
  • 适用场景：需要快速查找、插入、删除的键值对存储

Go 语言实现

4. 栈

原理介绍

栈是一种**后进先出（LIFO）**的数据结构，就像叠盘子。

  • 特点：只能在栈顶操作
  • 主要操作：push（入栈）、pop（出栈）、peek（查看栈顶）
  • 时间复杂度：O(1)
  • 适用场景：函数调用、括号匹配、表达式求值、DFS

5. 队列

原理介绍

队列是一种**先进先出（FIFO）**的数据结构，就像排队买票。

  • 特点：队尾入队，队首出队
  • 主要操作：enqueue（入队）、dequeue（出队）、front（查看队首）
  • 时间复杂度：O(1)
  • 适用场景：BFS、任务调度、消息队列

6. 二叉树

原理介绍

二叉树是一种树形数据结构，每个节点最多有两个子节点。

  • 组成：根节点、左子树、右子树
  • 遍历方式：
    • 前序：根 → 左 → 右
    • 中序：左 → 根 → 右
    • 后序：左 → 右 → 根
    • 层序：按层从上到下，从左到右
  • 适用场景：表达式树、文件系统、搜索

数据结构对比总结

数据结构 访问 插入 删除 空间 适用场景
数组 O(1) O(n) O(n) O(n) 随机访问
链表 O(n) O(1) O(1) O(n) 频繁增删
哈希表 O(1) O(1) O(1) O(n) 快速查找
栈 O(n) O(1) O(1) O(n) LIFO、回溯
队列 O(n) O(1) O(1) O(n) FIFO、BFS
二叉树 O(h) O(h) O(h) O(n) 层次结构

注：h 表示树的高度

小贴士

  1. 选择合适的数据结构：根据问题特点选择（频繁访问用数组，频繁增删用链表）
  2. 理解时间复杂度：知道每个操作的效率
  3. 掌握遍历方法：特别是树的各种遍历
  4. 多画图理解：画图能帮助理解指针操作
  5. 刷题时灵活运用：很多题目需要组合使用多种数据结构
